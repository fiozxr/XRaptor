XRaptor — Configuration Guide

Plain, clear steps to configure XRaptor for GitHub users

1. Where is the configuration file?

After installation, XRaptor settings live at:

/etc/xraptor/xraptor.conf

Open it with: sudo nano /etc/xraptor/xraptor.conf

2. Basic settings you can change

THRESHOLD

Number of failed logins allowed before blocking an IP.

THRESHOLD=5

Example: set to 3 → blocks after 3 failed attempts.

BAN_TIME

How long an IP is blocked (seconds):

BAN_TIME=3600

3600 = 1 hour. Use 600 for 10 minutes, 86400 for 24 hours.

LOG_FILE

Where XRaptor reads SSH failures from:

LOG_FILE="/var/log/auth.log"

STATE_FILE

Where XRaptor stores active bans (timestamp + IP):

STATE_FILE="/var/run/xraptor-state.txt"

WHITELIST

IPs that should never be blocked:

WHITELIST="/etc/xraptor/whitelist.txt"

To whitelist an IP:

echo "1.2.3.4" | sudo tee -a /etc/xraptor/whitelist.txt

3. Enable Telegram alerts

  1. Edit the config: sudo nano /etc/xraptor/xraptor.conf
  2. Set the fields:
ENABLE_TELEGRAM="yes"

TELEGRAM_BOT_TOKEN="your_token_here" TELEGRAM_CHAT_ID="your_chat_id_here"

Restart XRaptor after editing:

sudo systemctl restart xraptor

4. Enable Discord alerts

Edit the config:

ENABLE_DISCORD="yes"

DISCORD_WEBHOOK="https://discord.com/api/webhooks/...."

Restart service:

sudo systemctl.restart xraptor

5. Restart XRaptor after changes

sudo systemctl restart xraptor

Always restart to apply new settings.

6. Check XRaptor status

sudo systemctl status xraptor

If active, you will see active (running).

7. View logs

sudo journalctl -u xraptor -f

Shows live events: blocks, unblocks, alerts.

8. See currently blocked IPs

cat /var/run/xraptor-state.txt

Shows lines like: timestamp IP

9. Manually unblock an IP

Remove the IP line from the state file and restart:

sudo nano /var/run/xraptor-state.txt

remove the line for the IP

sudo systemctl restart xraptor

10. Troubleshooting — XRaptor isn’t blocking?

  • Make sure your SSH log exists: /var/log/auth.log or /var/log/secure.
  • XRaptor must run as root (systemd service runs as root).
  • Your system should use iptables or adapt the script for ufw/nftables.
  • Check threshold is not too high.
  • View journalctl -u xraptor -f for errors.

11. Uninstall XRaptor

sudo systemctl stop xraptor

sudo systemctl disable xraptor sudo rm /etc/systemd/system/xraptor.service sudo rm -rf /etc/xraptor sudo rm /usr/local/bin/xraptor.sh sudo systemctl daemon-reload

12. Quick summary

  • Edit config: /etc/xraptor/xraptor.conf
  • Restart: sudo systemctl restart xraptor
  • Whitelist admin IPs to avoid lockout
  • Enable alerts by adding tokens/webhooks
